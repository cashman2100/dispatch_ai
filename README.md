# 🏠 DispatchAI

**Извлечение знаний из звонков диспетчеров ЖКХ**

---

## Задача

Диспетчерские службы управляющих компаний (УК) ежедневно принимают сотни звонков от жителей. Диспетчеры — носители уникальных знаний о жилфонде, жителях, подрядчиках и истории проблем. Эти знания существуют только в их головах и теряются при увольнении.

В CRM [«Мой Дом Онлайн» (МДО)](https://мойдомонлайн.рф) сохраняются аудиозаписи звонков и транскрипции от голосового робота, но из них не извлекается никакой структурированной информации, кроме самой заявки. Годы диалогов — неиспользуемый актив.

## Для кого

- **Управляющие компании и ТСЖ** — клиенты МДО, которые хотят снизить зависимость от конкретных диспетчеров.
- **МДО как платформа** — получает конкурентное преимущество: от «тупого робота по скрипту» к «роботу, который знает ваши дома».

## Какая боль сейчас

1. **Знания заперты в людях.** Диспетчер знает, что «в доме на Ленина 12 лифт ломается каждый месяц, звонить нужно лифтовику Сергею». Уволился — знание потеряно.
2. **Робот работает по скрипту.** Голосовой робот Aimylogic не имеет контекста о доме или истории проблем.
3. **Нет способа накапливать опыт.** Сотни тысяч звонков за 9 лет лежат мёртвым грузом.

---

## Что делает PoC на демо

Берёт MP3-файл звонка и выдаёт структурированные факты:

1. **Диаризация.** Разделяет аудио на два голоса: диспетчер и житель (pyannote.audio).
2. **ASR.** Транскрибирует речь каждой роли отдельно (Whisper).
3. **Извлечение знаний.** LLM-агент анализирует речь диспетчера и извлекает факты по таксономии: жилфонд, сотрудники, история проблем (LLM + structured output).
4. **Хранение и поиск.** Факты сохраняются в SQLite с привязкой к адресу. CLI-команда: «покажи всё, что мы знаем про дом X».

```
MP3 ──▶ Diarization ──▶ ASR ──▶ Extraction Agent ──▶ SQLite
                        (pyannote)   (Whisper)   (LLM, JSON)      (факты по адресу)
```

## Что НЕ делает PoC (out-of-scope)

- Не заменяет голосового робота Aimylogic и не вмешивается в production-поток.
- Не обрабатывает звонки в реальном времени — только offline, batch.
- Не строит RAG-чатбот (возможное расширение).
- Не фильтрует PII автоматически (ручная модерация).
- Не дедуплицирует факты семантически (exact match по ключевым полям).
- Не интегрируется с CRM по API — данные берутся экспортом (папка MP3 + CSV).
- Не имеет веб-интерфейса — демо через CLI и Jupyter notebook.

## Стек

| Компонент | Технология |
|-----------|-----------|
| Диаризация | pyannote.audio 3.x |
| ASR | Whisper large-v3 |
| Extraction | LLM (Claude API / OpenAI) + structured JSON output |
| Хранение | SQLite |
| Оркестрация | Python (классы с методом `run()`) |
| Демо | CLI (Click/Typer) + Jupyter notebook |
| Язык | Python 3.11+ |

## Возможные расширения

1. Простой RAG — «задай вопрос по базе знаний дома».
2. Базовый PII-фильтр на regex (телефоны, паспорта).
3. Семантическая дедупликация через embeddings.
4. Обёртка оркестратора в LangGraph.
5. Простой веб-UI для просмотра фактов.
